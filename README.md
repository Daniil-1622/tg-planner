# tg-planner — Telegram-бот для личной продуктивности

Бот на Python для:
- расписания пар (ежедневный вопрос в 22:00 МСК);
- TODO-листа (задачи на сегодня/завтра, отметка выполненных);
- итогов дня (23:30 МСК);
- долгосрочных целей (чекап по пятницам 20:00 МСК + журнал прогресса).

Стек: Python 3.11+, `python-telegram-bot` (v20+, async), APScheduler, SQLite + SQLAlchemy, `python-dotenv`.

## Команды

- **`/start`** — приветствие и подсказка по возможностям
- **`/tasks`** — задачи на сегодня
- **`/done`** — отметить задачи выполненными (тот же список, что и `/tasks`)
- **`/add`** — добавить задачи на сегодня (списком, каждая с новой строки)
- **`/goals`** — список активных целей (с кнопками “выполнено/удалить/журнал”)
- **`/addgoal`** — добавить долгосрочную цель (название → дедлайн → мотивация)
- **`/goallog <точное название>`** — история записей по цели
- **`/schedule`** — показать выбранную пару на “завтра” (если уже выбрана)

## Как это работает (расписание)

Планировщик (Europe/Moscow):
- **каждый день 22:00** — “📚 К какой паре завтра?” (inline-кнопки 1–4 / нет пар) + карточка времени из `bot/config.py`
- **каждый день 23:30** — итог дня по задачам (если задачи есть)
- **каждую пятницу 20:00** — “📊 Еженедельный чекап целей” + ожидание текстового ответа (сохранится как запись в журнал целей)

## Настройка расписания пар

Открой `bot/config.py` и заполни словарь `PAR_SCHEDULE` под себя (времена вставать/собираться/выходить/начало пары).

## Переменные окружения

Бот читает переменные из окружения и (локально) из файла `bot/.env`.

Минимум:
- **`BOT_TOKEN`** — токен бота

Для рассылок по расписанию (22:00 / 23:30 / пятница):
- **`CHAT_ID`** — основной чат (обычно ваш личный чат с ботом)
- опционально **второй чат**: `CHAT_ID_2` (или `CHAT_ID2`)
- опционально **список чатов**: `CHAT_IDS=111,222` (через запятую или `;`)

Пример `bot/.env`:

```env
BOT_TOKEN=123456:ABCDEF...
CHAT_ID=111111111
# CHAT_ID_2=222222222
# CHAT_IDS=111111111,222222222
```

Важно: `bot/.env` **не коммитится**. Для примера есть `bot/.env.example`.

## Локальный запуск (Windows / PowerShell)

1) Перейди в папку бота:

```powershell
cd d:\tg-planner\bot
```

2) Установи зависимости:

```powershell
py -m pip install -r requirements.txt
```

3) Создай `bot/.env` (можно на основе `bot/.env.example`) и заполни `BOT_TOKEN` (+ `CHAT_ID`).

4) Запусти:

```powershell
py main.py
```

## Деплой на Railway

Проект лежит в подпапке `bot/`, поэтому в Railway:

1) Создай сервис из GitHub-репозитория.
2) В настройках сервиса укажи **Root Directory = `bot`**.
3) Тип сервиса — **Worker** (это long polling, веб-сервер не нужен).
4) Добавь Variables:
   - `BOT_TOKEN`
   - `CHAT_ID` (и при необходимости `CHAT_ID_2` / `CHAT_IDS`)
5) Нажми Redeploy.

Файл запуска: `bot/Procfile` (`worker: python main.py`).

## Как получить BOT_TOKEN

1) Открой [@BotFather](https://t.me/BotFather)
2) `/newbot` → создай бота
3) Скопируй токен и вставь в `BOT_TOKEN`

## Как получить CHAT_ID

Самый простой способ:
1) Напиши боту любое сообщение в личке
2) Открой в браузере:

`https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/getUpdates`

3) В ответе найди `chat.id` — это число и есть `CHAT_ID`
